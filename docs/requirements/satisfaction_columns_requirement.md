# 家計簿テーブル 満足度列 要件定義 (v0.1)

## 目的
- 家計簿の各支出行に対して、ユーザー自身の主観的評価を「上位満足度 (Top3)」または「下位満足度 (Worst3)」として記録し、その理由メモを残せるようにする。
- 家計の見直しやサブスク解約検討時に、支出の価値判断を後から参照できる状態を作る。

## スコープ
- 対象画面: `https://moneyforward.com/cf` の家計簿テーブル。
- 対象行: `is_target = 1` の行を主対象とするが、収入/振替を含む全行で UI は表示可能とする（ユーザー判断で選択）。
- ブラウザ拡張側で列を追加し、MoneyForward 本体のデータは変更しない。

## 非スコープ
- サーバー同期やエクスポート機能（現時点ではローカル保存のみ）。
- 集計・ランキングの自動生成（当面は記録と表示に限定）。

## UI 要件
- テーブル右端に2列を追加する。
  - **満足度** 列: セレクトボックスで7択
    - 未選択
    - Top1 / Top2 / Top3
    - Worst1 / Worst2 / Worst3
  - **満足度メモ** 列: テキスト入力（単一行、最大 ~120 文字想定）。
- 各セルは1要素のみ（1列目=セレクト、2列目=メモ）。セレクトとメモを同一セルに入れない。
- 選択・入力は行内で完結し、行クリック等の既存操作と干渉しないようイベント伝播を止める。
- キーボード操作・スクリーンリーダー対応のため、`aria-label` を各要素に付与する。

## データモデル
- 保存先: `chrome.storage.local`。
- キー: 既存 `TxKey = "tx:${transactionId}"` を流用。
- 値: `{ rank: "top1"|"top2"|"top3"|"worst1"|"worst2"|"worst3"|null, note: string }`。
- バージョン: 今回は v1。将来の拡張に備えフィールド追加を許容する。

## エラーハンドリング
- storage 書き込み失敗時は UI 上でトーストまたはインラインエラーを表示し、ユーザー入力を破棄しない。
- 取引ID取得不可行はスキップし、UIを描画しない（誤保存防止）。

## パフォーマンス
- MutationObserver のデバウンスチェーンに統合し、DOM再描画時に再挿入する。1行あたり軽量な select/input のみとし、100〜300行規模でも体感性能を維持する。

## セキュリティ・プライバシー
- データはローカルにのみ保存し、外部送信しない。
- メモはサニタイズ不要なテキスト入力のみを扱い、HTML挿入を避ける。

## 受け入れ条件 (Acceptance Criteria)
- 家計簿テーブルに「満足度」「満足度メモ」2列が表示される。
- 各行で Top/Worst ランクを選択し、理由メモを入力・保存・復元できる。
- 列追加後も既存機能（ラベル選択、Gemini ハイライト、二重計上警告、カテゴリ警告）が動作し、店名/金額抽出が壊れない。
- 収入行や対象外行でも UI 表示でエラーなく操作可能。
