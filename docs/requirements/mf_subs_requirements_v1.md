# マネーフォワードME拡張機能 — サブスク検出・ラベル管理 要件定義書 (v1)

## 1. 概要
- 家計簿画面「月次の収支明細」からサブスクの可能性がある支出を抽出し、ユーザーが手動で利用状態ラベルを管理できる拡張機能。
- ユーザーが一度付与した「店名＋金額」組み合わせを翌月以降に自動適用する。
- Gemini API は **ハイライト色の判定のみ** に利用し、ラベル値へは一切影響させない。

## 2. 対象画面とDOM前提
- 対象URL: `https://moneyforward.com/cf`
- 取引行: `<tr class="transaction_list ...">`
- 取引ID: 行内 `<input name="user_asset_act[id]">` の値（例: `1788783…`）。行識別の主キーとして使用。

## 3. ラベル機能（ユーザー操作）
### 3.0 表示位置（サブスク用ラベル列）
- メモ列とは別の専用列を家計簿テーブルに追加し、メモ欄の直後に配置する。
- 満足度機能（任意）よりも左側に固定し、満足度OFF時の列削除処理で巻き込まれないこと。
- 既存メモ文字列の表示や編集は従来のメモ列に残し、ラベル列は `<select>` のみを持つ。

### 3.1 ラベル種類
- 利用中 (`using`)
- ほぼ未使用 (`rarely`)
- 解約予定 (`cancel`)
- 未設定 (`""`)

### 3.2 ラベルUI
- 各行の「ラベル専用列」に `<select>` を埋め込む（メモ列から分離）。
- 初期値は自動判定結果を適用。
- ユーザー変更は即時保存。

### 3.3 満足度UI（拡張）
- 各行末尾に「満足度」「満足度メモ」を配置する。
- 保存キーは **取引ID／店名＋金額＋日付** の両方を保持し、どちらかで復元できるようにする。
  - 取引IDキー: `tx:{transactionId}`
  - 店名＋金額＋日付キー: `sd:{normalizedStore}|{amount}|{date}`
  - 正規化と金額絶対値はサブスクラベルと同一ルール。
  - 日付は YYYY-MM-DD を想定。

## 4. ラベル保存仕様
### 4.1 優先度1: 取引ID → ラベル
- キー: 取引ID
- 値: ラベル (`using/rarely/cancel`)
- 初期値適用時は最優先で使用。

### 4.2 優先度2: 店名＋金額 → ラベル
- キー: `"店名|金額"`（店名は正規化後）
- 値: ラベル (`using/rarely/cancel`)
- 翌月以降の再出現に自動適用。

### 4.3 店名正規化ルール
1. 前後の空白を削除
2. 連続空白を単一スペースへ統一
3. 絵文字・不可視文字を削除
4. 全角/半角は変換しない（重要）

### 4.4 ラベル選択時の処理
- 取引ID→ラベル を保存
- 店名＋金額→ラベル も保存
- 行を黄色ハイライト

### 4.5 ラベル解除時の処理
- 未設定 (`""`) に戻した場合：
  - 行ハイライトを削除
  - 取引ID保存データを削除
  - 店名＋金額保存データも削除
  - 以後の自動適用を停止

## 5. Gemini 判定機能
### 5.1 自動実行タイミング（1回/月）
- `/cf` ページ表示
- 月切替後のページ再描画
- 再実行防止フラグ: `sessionStorage.setItem("mf_subs_checked_YYYY-MM", true)`

### 5.2 用途（ラベルには反映しない）
- score >= しきい値 → 薄い黄色でハイライト
- score < しきい値 → 何もしない
- ラベル値への反映は禁止。

### 5.3 Gemini 入力例（JSON）
```json
{
  "month": "2025-11",
  "transactions": [
    {
      "id": "1788783355412939428",
      "date": "2025-11-18",
      "store": "ＵＳＥＮ NETWORKS",
      "amount": 4950,
      "category": "通信費",
      "subcategory": "固定費",
      "memo": "ネット回線",
      "payment_source": "楽天カード"
    }
  ]
}
```

### 5.4 Gemini 出力例（JSON）
```json
{
  "results": [
    {
      "id": "1788783355412939428",
      "score": 82,
      "service_name": "USEN インターネット",
      "reason": "通信系で金額が一定であるため"
    }
  ]
}
```

※ 使用するのは `score` のみ。

## 6. 候補抽出の前処理
Gemini に送る前に以下を除外し、残りを「候補」とする：
- 振替
- 投資積立
- 住宅ローン
- その他見直し対象外の固定費（固定ロジック）

## 7. 背景色ハイライト
- 条件:
  - Gemini `score` が閾値（初期値70）以上 **または** ラベルが設定済み
- 色: 薄い黄色
- CSS例:
```css
.mf-sub-highlight {
  background-color: rgba(255, 255, 0, 0.25);
}
```

## 8. オプション設定画面
- 設定項目:
  - Gemini API キー
  - サブスクしきい値（0〜100）
  - （将来拡張）除外カテゴリ設定
  - Gemini解析の有効/無効トグル（初期値: 有効）
  - サブスク列の表示/非表示トグル（初期値: 表示）
- 保存先: `chrome.storage.local` または `chrome.storage.sync`

### 8.1 機能トグル要件
- トグルが OFF の場合、Gemini 解析呼び出しは行わない。
- OFF 時は Popup の「このページで再解析」ボタンを非活性化し、ホバー/フォーカス時に無効理由を表示する（例: `解析は無効化されています`）。
- トグル状態は `settings` に保存し、オプション画面と Popup で同じ値を参照する。

## 9. エラー時の動作
- API キー未設定 / Gemini API エラー / 通信エラー発生時：
  - 解析をスキップ
  - 手動ラベル機能は常に利用可能
  - 必要に応じて画面上部に控えめなメッセージを表示

## 10. 全体処理フロー（最終版）
1. `/cf` ページ読み込み
2. DOM検知後、各行へラベル初期値を適用
   - 取引ID→ラベル
   - 店名＋金額→ラベル
3. 未解析なら Gemini へ候補行を送信
4. `score` に応じて黄色ハイライト
5. ユーザーがラベル設定
   - 取引ID保存
   - 店名＋金額保存
   - 行を黄色化
6. ラベル解除時
   - 黄色解除
   - 保存データ削除
   - 自動適用を停止
