# MoneyForward Toolpack — 大項目×中項目ルール警告 要件 (v0.1)

## 目的と背景
- 家計簿行に設定された「大項目」「中項目」の組み合わせ不整合を視覚的に警告し、分類ミスを早期に発見できるようにする。
- 既存のサブスク可視化・重複警告と共存し、ユーザーの手動修正を補助することを目的とする。自動修正は行わない。

## スコープ
- 対象画面: MoneyForward ME の家計簿ページ `/cf` テーブル。
- 対象行: `is_target=1` かつ支出行（負の金額）に限定。
- 対象カテゴリ: テーブル上に表示される大項目（`.v_l_ctg`）と中項目（`.v_m_ctg`）。
- リスト管理: ブラウザ拡張の設定としてホワイトリスト／ブラックリストを保持し、ローカルまたは sync ストレージに保存する。

## 非スコープ
- MoneyForward 本体のカテゴリマスタの変更や同期。
- 自動でカテゴリを修正・置換する処理。
- API 連携やバックエンド保存。

## ユースケース
- 大項目「食費」に対し中項目が「家賃」など誤分類されている場合に即時警告する。
- 運用ルールで許容組み合わせが限定されているチーム利用時に、ホワイトリストで強制的にチェックする。
- よくある誤りの組み合わせをブラックリスト登録して再発防止する。

## 機能要件
- ホワイトリスト
  - 登録されている組み合わせのみ許可し、それ以外は警告とする。
  - ホワイトリストが空の場合は「無効」とみなし、この要件による警告は発生させない。
- ブラックリスト
  - 登録された組み合わせを検出した場合に警告とする。
  - ブラックリストが空の場合は「無効」とみなす。
- 判定優先度
  - 両リストが設定されている場合はホワイトリスト判定を先に行い、ホワイトリストで許可された組み合わせはブラックリストに載っていても警告しない（明示的許可を優先）。
- 表示要件
  - 警告対象となった行の大項目セル・中項目セルを警告色でハイライトし、視認性を確保する。
  - 行全体のハイライト（サブスク強調・重複警告など）がある場合でも警告色が判別できるよう配色を設計する。
  - ツールチップなどで「カテゴリ組み合わせがルール外である」ことを短文で提示する。
- 設定・操作
  - オプション画面でホワイトリスト／ブラックリストを登録・編集・削除できる UI を提供する。
  - フィーチャートグル: `categoryRuleAlertEnabled`（仮称）を追加し、機能を一括で ON/OFF できる。
- 更新タイミング
  - 初期ロードと MutationObserver 経由の DOM 変化時に最新ルールを適用する。
  - ルール変更（オプション保存）後は家計簿タブで即時またはリロード後に反映される。

## データ要件
- ルールは `{ large: string, middle: string }` の配列として保存する。キーは前後の全角・半角スペースを除去した文字列を採用。
- 大項目のみ・中項目のみの部分指定はサポートしない（両方必須）。
- 保存容量は sync 上限を考慮し、最大エントリ数を 200 件程度に制限する（超過時は保存エラーを返す）。

## エッジケース・例外
- 大項目または中項目が空文字・`未分類` 表記の行は警告判定から除外する（誤検知抑止）。
- 文字の全角／半角差、前後の空白、絵文字は正規化して判定する。
- 家計簿 UI 変更でクラス名や DOM 構造が変わった場合、判定処理は安全に失敗（警告しない）し、コンソールにデバッグログを出す。

## 非機能要件
- テーブル 2,000 行程度でも体感遅延を増やさない。1 行あたり O(1) 判定で総 O(n)。
- オフラインでも動作し、設定読み込みに失敗した場合は警告を出さずフェイルセーフとする。

## 依存・影響範囲
- `src/content/index.js` の家計簿 DOM 抽出ロジックと `featureFlags` に新設定を追加する。
- 既存のサブスクハイライト（黄色）・重複警告（アンバー）との視覚衝突を避けるため、スタイル優先順位を調整する必要がある。
