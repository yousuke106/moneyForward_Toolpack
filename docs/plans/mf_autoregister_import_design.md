# MoneyForward Toolpack — CSV自動入力機能 移植設計メモ（家計簿画面ボタン統合）

## 背景・目的
- 既存 `tmp/moneyforwardAutoRegister` のポップアップ起動型 CSV 自動入力機能を、家計簿画面 `/cf` で直接使えるよう統合する。
- 「手入力」ボタン右隣に「CSV取込み」ボタンを追加し、ユーザーの文脈内でファイル選択〜自動入力を完結させる。

## 対象DOM（挿入位置）
- 親: `div.pull-right.mf-mb-medium`
- 既存ボタン: `button.cf-new-btn.btn.modal-switch.btn-warning`（手入力）
- 追加要素: `button.mf-csv-import-btn`（セカンダリ色、幅100〜120px程度）と隠し `<input type="file" accept=".csv">`
- フラグ: 重複挿入防止用に親へ `data-mf-csv-import="1"` を付与。

## 想定フロー
1. ボタン追加（content script 初期化時、ページURLが `/cf` のときのみ）
2. クリックで file input を開く → CSV読込 → パース
3. ヘッダ検証（`取引日` `出金金額（円）` `取引先` `取引内容` 必須）
4. レコード整形（送金/支払の前置詞ロジックは既存どおり）
5. 手入力モーダルを開く（既存「手入力」ボタン click）
6. 日付/金額/内容を入力 → 送信 → 確認 → 次レコード…を直列で処理
7. 完了/途中エラーをステータス表示（モーダル隣にインラインメッセージ、alert は廃止）

## エラーハンドリング/UX
- ヘッダ不足・空行・0件 → ボタン横にエラーテキスト表示
- DOMセレクタ未検出（手入力ボタン・入力フィールド） → エラーステータス表示して中断
- 送信/確認ボタンが無い場合は3回までリトライ（500ms間隔）後に中断
- 進捗表示: 「n/N 処理中...」「完了」「エラー: ...」をテキストで表示
- キャンセル: 取込み開始後に「停止」リンクを同じ領域に表示し、キューをクリア

## バリデーション・制約
- 最大行数: 200件（超過で警告し、先頭200件のみ処理）
- 日付: `YYYY-MM-DD` 形式に正規化、10文字未満はエラー
- 金額: 数値化失敗でエラー。負号はそのまま。
- 取引先/内容: 255文字上限で truncate（XSS防止のため文字列は textContent 経由で設定）

## リスク・留意点
- DOM変更への耐性が低いのでセレクタを一箇所に集約し、未検出時は静かに諦める。
- 連続送信によるサーバ側バリデーション・レート制限の可能性があるため、最小限の待機とリトライ回数に留める。
- ファイル入力はユーザージェスチャ前提のため、バックグラウンド起動は不可（必ずクリックから開始）。
