# マネーフォワードME拡張 — 二重計上チェック詳細設計（v0.1）

## 目的と背景
- 複数口座を紐づけた際に同一取引が重複して登録されるケースを検知し、テーブル上で「警告色」による視覚的な注意喚起を行う。
- 本機能はデータの自動削除・統合は行わず、ユーザーが手動で確認・修正するための補助を目的とする。

## スコープ
- 対象画面: `https://moneyforward.com/cf` の家計簿テーブル。
- 対象データ: `is_target=1` の支出行（`is_income=1` は除外）で、金額がマイナス表記（支出）かつパース可能なもの。
- 本機能は Gemini 解析やサブスクラベル付与と併用可能とし、既存ハイライトと共存させる。
- 非スコープ: 重複データの自動マージ、メモ/カテゴリの自動変更、API/サーバー側への反映。

## 判定条件（厳密一致）
- **キー要素**: `date (YYYY-MM-DD)`, `store (正規化後)`, `amount (絶対値・整数)`.
- 同一キーで **2件以上** 存在する行を「二重計上疑い」としてマーキングする。
- 店名正規化は既存 `normalizeStoreName` を再利用（trim → 連続空白畳み → 絵文字/不可視文字除去・全角保持）。
- 金額は既存 `parseAmount` 相当でカンマ除去・Unicodeマイナス対応後、絶対値化して整数扱い。
- 日付は `parseDate` から取得した `YYYY-MM-DD` をそのまま使用（時刻・タイムゾーン差分は考慮しない）。

## 抽出とフィルタリング
- ベースは `collectTransactions` と同じ DOM 抽出ロジックを用いるが、以下を追加/調整する:
  - `is_income=1` を除外（重複は主に支出で発生する想定）。
  - `is_target=1` のみ対象。
  - `amount`, `store`, `date`, `id` のいずれかが欠損した行はスキップ。
  - `store` が空文字の場合は重複判定対象外（誤判定抑止）。
  - 既存の除外キーワード（振替/投資積立/住宅ローン/固定費）も除外を継続（要件 v1 と整合）。

## キー生成とデータ構造
- **DuplicateKey 形式**: `dup:${date}|${normalizedStore}|${amount}`。
- `Map<DuplicateKey, { txIds: string[] }>` を作成し、`txIds.length >= 2` をヒットとする。
- DOM への反映用に `Map<string /*txId*/, HTMLTableRowElement>` を構築しておき、ヒットした txId 群に対してクラス付与を行う。
- 将来のテスト容易性のため、キー生成・グルーピング処理は `src/data` もしくは `src/lib` に純粋関数として切り出す。

## UI/スタイル仕様
- 新規クラス: `.mf-sub-duplicate`
  - 背景色: 警告系の淡いアンバー `rgba(240, 140, 0, 0.20)`。
  - 左ボーダー: `4px solid #F08C00`（視認性と既存黄色ハイライトとの差別化）。
  - テーブルセルにも同色を適用するため、`.mf-sub-duplicate td` に同系の背景を付与。
- 既存 `.mf-sub-highlight`（薄い黄色）との共存ルール:
  - 双方が付与される場合は `.mf-sub-duplicate` を優先する（より強い警告のため）。CSS では `.mf-sub-duplicate.mf-sub-highlight` に対しボーダー維持＋背景をアンバー寄りに固定。
  - スタイルは CSS の優先度で制御し、JS 側ではクラスの付け外し順に依存しない。
- 視覚ヒント: 行ホバー時に `title="同日・同内容・同額の取引が複数あります"` を付与して理由を提示（アクセシビリティ補完）。

## 処理フロー
1. DOM 解析（既存の `scheduleInit` 後、または同タイミング）で全取引行を走査。
2. 抽出・フィルタリング後、`DuplicateKey` ごとにグループ化。
3. `txIds.length >= 2` のグループに含まれる行へ `.mf-sub-duplicate` を付与し、非ヒット行からは当該クラスを除去。
4. DOM 変化（MutationObserver）でもデバウンス実行し、月切替やフィルタ操作で最新状態を反映。
5. ユーザー操作（ラベル変更）とは独立して動作。ラベル選択で `.mf-sub-highlight` が付与されても、重複警告があれば `.mf-sub-duplicate` が残る。

## 設定・トグル
- 既存 `settings.featureFlags` に `duplicateCheckEnabled: boolean` を追加（デフォルト `true`）。
- Options 画面の拡張は別タスクで実装予定。トグル OFF 時はクラス付与処理をスキップし、既存警告クラスも除去する。

## エッジケースと誤検知抑止
- 一括決済・サブスクリプションで同日に同額が複数存在する場合（例: 複数サービスの同額決済）は誤検知となりうる。初期リリースでは警告のみとし、ユーザー判断に委ねる。
- 日付跨ぎ（T+1 記帳）パターンは要件外のため扱わない。必要なら ±1日のゆるい判定を将来オプション化する。
- 金額0円や店名空は除外。

## テスト観点（追加分）
- グルーピング:
  - 同日・同店名（正規化後）・同額で2件 → 2件ともヒット。
  - 片方が店名末尾に絵文字 → 正規化後一致しヒット。
  - 金額が `-2,860` と `2860` → 絶対値化で一致しヒット。
  - 日付が `2025-11-01` と `2025/11/01` → 前者のみ採用（パース結果に依存）。
  - store が空文字 → 非ヒット。
- DOM 付与:
  - ヒット行に `.mf-sub-duplicate` が付き、非ヒット行には付かない。
  - `.mf-sub-duplicate` と `.mf-sub-highlight` の両方が付いた場合にアンバー背景が保持される。
- トグル:
  - `duplicateCheckEnabled=false` でクラス付与が行われない。
  - トグルを OFF→ON に変更すると最新 DOM を再評価して付与される。
