# オプション画面ワイヤーフレーム（v1） — MF Subscription Checker

## 目的
Chrome 拡張のオプションページで、APIキー・サブスク判定しきい値・モデル選択を直感的かつ制約内で入力できるUIを定義する。

## 画面構成（1カラム）
- **ヘッダ**: タイトル「設定」＋サブテキスト「データは同期上限超過時にローカルへ退避します」。
- **フォームセクション**
  1. **Gemini APIキー（必須）**
     - テキスト入力、幅100%。
     - プレースホルダ: `AIza...` など鍵を示唆する例は避け、空にする。
     - バリデーション: 非空チェック。未入力時は下部に赤字「APIキーを入力してください」。
  2. **サブスクしきい値 (0–100)**
     - type=number、step=1、デフォルト70。
     - バリデーション: 未入力・範囲外時は「0〜100の整数を入力してください」。
  3. **モデル選択**
     - セレクト: `gemini-2.0-flash`（推奨） / `gemini-2.0-pro`。
     - 補足文: 「費用・速度に応じて選択」。
  4. **保存ボタン**
     - プライマリ色（例: #00695c）。disabled状態: 入力不備時。
- **ステータス表示**
  - 成功: 「保存しました: sync」「保存しました: local（sync容量超過のため）」。
  - 失敗: 赤字でエラー本文を表示。
- **フッタの簡易リンク**
  - 「拡張の権限/保存先について」→ 短い説明ツールチップまたは別途モーダルは不要、同ページ下部に段落で説明。

## 状態遷移
1. 画面ロード
   - `storage.sync` → `storage.local` の順で設定取得し、フォームに反映。
   - ロード元をステータスに表示（例: sync / local / 未保存）。
2. 入力
   - 任意フィールド変更で「保存」活性化。
3. 保存
   - `saveSettingsWithFallback` を呼び、結果に応じてステータス文言を更新。
   - syncしきい値超過 or 書込エラー時は local へ自動退避し、その旨を表示。
4. バリデーションエラー
   - フィールド直下にエラーテキスト、保存ボタンは disabled。

## 文言案
- 成功: 「保存しました: sync」「保存しました: local（同期上限超過のためローカル保存）」。
- 失敗: 「保存に失敗しました: <error.message>」。
- 未保存: 「未保存です。設定を入力してください」。
- ロード元: 「ロード元: sync」「ロード元: local」。

## アクセシビリティ
- 各 input/select に対応する `<label>` を紐付け（`for` 属性）。
- 保存ボタン: キーボードで操作可能、フォーカスリングを維持。
- ステータス領域に `aria-live="polite"` を付与して変更を通知。

## レイアウト/スタイル指針
- 余白: 上下24px、入力間12px。
- フォント: system UI系、14px基準。
- 色: 成功 #00695c、エラー #b00020。
- モバイル幅（320px）でも横スクロールが出ない 1カラム構成。

## 今後の拡張プレースホルダ
- 除外カテゴリの複数選択（タグ入力）を下部に追加できるスペースを確保。
- 「データの保存先を選択（sync/local）」トグルを追加する場合はステータス行の直上に配置。
