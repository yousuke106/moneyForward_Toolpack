# 大項目×中項目ルール警告 チェックリスト (v0.1)

## 前提
- 拡張の設定で `categoryRuleAlertEnabled` が ON。
- ホワイトリスト／ブラックリストのテスト用データを事前に登録。
- 対象ページ: `https://moneyforward.com/cf` の家計簿テーブルが表示されている。

## 表示・ハイライト
- [ ] ホワイトリスト未登録の組み合わせを含む行で、大項目セルと中項目セルが警告色になり、title に「ホワイトリスト未登録」と表示される。
- [ ] ホワイトリストに登録されている組み合わせでは警告が出ない。
- [ ] ブラックリスト登録の組み合わせで警告色になり、title に「ブラックリスト対象」と表示される。
- [ ] ホワイトリストとブラックリスト双方に同一組み合わせがある場合、警告が出ない（ホワイトリスト優先）。
- [ ] サブスク黄色ハイライト・重複アンバーハイライトと重なっても警告色が視認できる。

## 判定ロジック・エッジケース
- [ ] 大項目または中項目が空文字／`未分類` の行は警告されない。
- [ ] 前後空白や全角スペースを含むカテゴリでも正規化され、一致判定される。
- [ ] ルールが 0 件の場合、警告は一切表示されない。
- [ ] フィーチャートグルを OFF にすると既存の警告クラスがすべて除去される。

## 設定・永続化
- [ ] オプション画面でルールを追加・削除し、家計簿タブをリロードすると反映される。
- [ ] 200 件超の保存を試みるとエラーメッセージが表示され保存されない。
- [ ] sync 保存不可時に local 保存へフォールバックし、ステータス表示が適切に切り替わる。
- [ ] ホワイト/ブラックのタブ切替でそれぞれ独立したリストと入力欄が表示され、切替で内容が入れ替わる。
- [ ] 入力行で大項目・中項目が空のまま追加しようとするとエラー表示される。
- [ ] 重複する組み合わせを追加しようとすると「既に登録されています」等のエラーが出て登録されない。
- [ ] 削除アイコンで単一行を削除でき、保存後の家計簿画面で警告が消える。

## パフォーマンス・安定性
- [ ] 1,000 行以上表示時でも目視で描画遅延が増えていない。
- [ ] ページ切替やフィルタ適用で MutationObserver が再適用され、警告が最新状態に保たれる。
