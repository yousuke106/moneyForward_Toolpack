# 家計簿ダウンロード（右クリックメニュー）移植チェックリスト

## 事前準備
- [ ] manifest に `downloads`/`contextMenus`/`scripting`/`tabs` と `https://moneyforward.com/cf/csv*` などの host_permissions が含まれている。
- [ ] 拡張をアンパックで読み込み、`chrome://extensions` で「更新」を実行。
- [ ] 家計簿ページ `https://moneyforward.com/cf` を表示し、ページヘッダーが読み込まれている。

## 機能確認
- [ ] ページ上で右クリックすると「表示中の年月の家計簿CSVをダウンロード」メニューが表示される。
- [ ] メニュー選択でバッジが `DL` → 数秒後に消える。ダウンロードダイアログが表示され、ファイル名が `moneyforward_YYYYMM.csv` になっている。
- [ ] ダウンロード後、`chrome.storage.sync` の `mf_month_year` が表示中の年月で更新され、ポップアップを開いたときに同値で初期化される。
- [ ] ログインしていない状態でメニューを押すと、バッジが `NG` など失敗表示になり、不要なダウンロードは走らない（ログインタブ自動オープンはしない）。
- [ ] 家計簿ページ以外（例: オプション画面や他サイト）で実行すると `NA` または `NG` バッジになり、不要なダウンロードが走らない。
- [ ] MoneyForward 側のヘッダーDOMが取得できない場合（意図的にセレクタを外す等）に `NA/NG` が表示され、例外でSWが落ちない。
- [ ] オプションのトグルを OFF にするとコンテキストメニューが非表示または無効化され、ON に戻すと即時有効化される。

## 競合・回帰確認
- [ ] 既存の Gemini 解析・ラベル表示が正常に動く（新規権限追加や SW 拡張による影響なし）。
- [ ] downloads API の `onDeterminingFilename` で他機能のダウンロードに影響していない（他URLの保存でファイル名が書き換わらない）。
- [ ] バッジ表示が他の拡張機能や既存処理と競合しない（表示/クリアのタイマーが意図通り）。

## コード品質チェック（カテゴリ別）
- [ ] 背景SW: 例外は catch でバッジへ集約、不要リスナーやタイマーが残っていない。
- [ ] コンテキストメニュー: ID/タイトルが定数化され、トグル変更時に登録/削除が冪等。
- [ ] URL生成・月抽出: 正規表現とパディングが純粋関数化されテスト済み。異常値で早期リターン。
- [ ] ダウンロード処理: `onDeterminingFilename` が自拡張発のみを対象にし、他ダウンロードを汚染しない。
- [ ] 設定・トグル: オプションUIと storage スキーマが一致し、`storage.onChanged` で即時反映される。
- [ ] 権限: manifest の permissions/host_permissions が最小で、README/ストア説明に追加理由を記載。
